# Plano: Tornar módulo Resend opcional

## Problema
O app crasha ao iniciar quando `RESEND_API_KEY` está vazio. O bug está no `nestjs-resend` v1.1.0: o construtor de `ResendService` faz `return` antes de chamar `super()` quando `apiKey` é falsy, causando `ReferenceError`.

## Solução
Remover a dependência de `nestjs-resend` e usar o pacote `resend` diretamente, com tratamento para API key ausente.

## Arquivos a modificar

### 1. `apps/auth/src/mail/mail.module.ts`
- Remover import do `ResendModule` do `nestjs-resend`
- Criar um provider customizado que instancia `Resend` (do pacote `resend`) apenas se `RESEND_API_KEY` estiver definido
- Se não estiver, injetar `null` como `ResendClient`

### 2. `apps/auth/src/mail/mail.service.ts`
- Trocar `ResendService` (do nestjs-resend) por uma interface/primitiva do `resend` (`Resend` class)
- No construtor, receber o client (`Resend | null`)
- Em `sendVerificationEmail`, checar se o client existe antes de usar; logar aviso e retornar sem erro quando a key não estiver configurada

## Verificação

2. Verificar que o app inicia sem erros nos logs
3. Confirmar que endpoints de auth funcionam (cadastro, login)
