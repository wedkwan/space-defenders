# Plano: Sistema de Matchmaking para Multiplayer

## Contexto

Atualmente o multiplayer cria salas fixas onde jogadores entram/saem livremente. Isso causa problema de "farm de vida" (sair e voltar para recuperar HP). O fluxo precisa ser: **Multiplayer → Tela de Matchmaking (esperando 2 jogadores) → Partida começa**.

---

## Fluxo novo

```
Play → Clique Multiplayer → /play/multiplayer
  → Tela de Matchmaking (loading, "1/2 jogadores")
  → 2 jogadores prontos → Servidor cria sala e inicia jogo
  → Tela do jogo (canvas, gameplay)
  → Fim de jogo → Botão "Voltar ao Menu"
  → Se sair durante matchmaking → remove da fila
```

---

## Alterações no Servidor

### `apps/game-server/src/game/game.gateway.ts`

Adicionar fila de matchmaking e novos handlers:

```typescript
// Fila: Map<socketId, { playerId, name }>
private matchmakingQueue: Map<string, { playerId: string; name: string }> = new Map();
```

**Novos eventos:**

| Evento | Direção | Payload | Descrição |
|--------|---------|---------|-----------|
| `matchmaking:join` | Client→Server | `{ playerId, name }` | Entra na fila |
| `matchmaking:leave` | Client→Server | `{}` | Sai da fila |
| `matchmaking:status` | Server→Client | `{ count: number, total: 2 }` | Atualiza contagem |
| `matchmaking:matched` | Server→Client | `{ roomId, playerIndex }` | Partida encontrada |

**Lógica:**
1. `matchmaking:join`: Adiciona na fila. Se fila tem 2, cria sala, adiciona ambos, emite `matchmaking:matched` para cada um, emite `game:players` com nomes, limpa fila.
2. `matchmaking:join` (fila 1): Emite `matchmaking:status` com `{ count: 1, total: 2 }`.
3. `matchmaking:leave`: Remove da fila. Emite `matchmaking:status` atualizado para quem ficou.
4. `handleDisconnect`: Se o socket estava na fila, remove e atualiza contagem.
5. **Início do jogo**: Quando 2 jogadores estão na sala, `game-room.ts` já inicia automaticamente (`players.size >= 1 && status === 'waiting'`). Basta ajustar para `players.size >= 2`.

### `apps/game-server/src/game/game-room.ts`

Mudar condição de início:
```typescript
// Antes: inicia com 1 jogador
if (this.players.size >= 1 && this.status === 'waiting') {

// Agora: espera 2 jogadores
if (this.players.size >= 2 && this.status === 'waiting') {
```

## Alterações no Cliente

### `apps/web/src/hooks/useGameEngine.ts`

O hook precisa de dois modos de operação:

1. **Modo matchmaking**: Conecta ao servidor, emite `matchmaking:join`, escuta `matchmaking:status` e `matchmaking:matched`.
2. **Modo jogo**: Após `matchmaking:matched`, emite `game:join` com o roomId recebido, entra no jogo.

Adicionar ao hook:
- `matchmakingStatus: { count: number, total: number } | null`
- `isMatched: boolean`
- `joinMatchmaking()` — inicia o processo
- `leaveMatchmaking()` — sai da fila

### `apps/web/src/app/play/multiplayer/page.tsx`

A page multiplayer agora tem dois estados:

1. **Matchmaking** (inicial): Tela de loading com animação, contagem "1/2 jogadores", botão "Cancelar"
2. **Jogo** (após match): Canvas, gameplay (código existente)

Transição: quando `isMatchmued` vira `true`, troca o render de matchmaking para o canvas.

### Design da tela de matchmaking

Tela preta com:
- Logo/título do jogo
- Ícone de nave animado (pulsando ou girando)
- Texto "PROCURANDO ADVERSÁRIO..."
- Contador grande "1 / 2"
- Texto "PILOTOS ENCONTRADOS"
